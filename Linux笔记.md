# Linux笔记

## 怎么更方便地对服务器上计算出来的数据进行分析

### 1.vscode方案

#### 1. 安装 ==Remote - SSH== 插件

1. 打开 VS Code，点击左侧边栏的 **扩展 (Extensions)** 图标（或者按 `Ctrl+Shift+X`）。
2. 在搜索框输入 ==`Remote - SSH`==。
3. 找到由 Microsoft 出品的插件并点击 **安装 (Install)**。

------

#### 2. 连接到服务器

1. 安装完成后，VS Code 左下角会出现一个蓝色的 **“><”** 形状的小图标，点击它。
2. 在弹出的菜单中选择 **Connect to Host... (连接到主机...)**。
3. 选择 **Add New SSH Host... (添加新的 SSH 主机...)**。
4. 输入你的连接命令，格式为：`ssh 用户名@服务器IP地址`（例如：`ssh root@192.168.1.100`）。
5. 按照提示选择保存配置文件的位置（默认第一个即可），然后右下角会弹出“Host added”，点击 **Connect**。
6. 根据提示输入服务器账号的 **密码**。

------

#### 3. 打开服务器文件夹

1. 连接成功后，点击左侧边栏的 **文件资源管理器 (Explorer)**。
2. 点击 **Open Folder (打开文件夹)**。
3. 这时你会发现路径已经变成了服务器上的目录系统。选择你存放数据和代码的文件夹（例如 `/home/yourname/project`），点击确定。
4. 再次输入密码确认。

> ==由于我们服务器的架构比较老，与现在的vscode中的ssh不兼容，无法满足现在利用vscode中的ssh插件的基本要求，这种在这里不适用，只能另寻他法。==

### 2.jupyterlab方案

> 这是目前做数据分析最流行的方式。你不需要 VS Code，只需要服务器上的 Python 环境和一个浏览器。

> 一些代码：
>
> `pip3 install --user jupyterlab`这个代表在服务器上安装jupyterlab。这里的–user的意思是：把软件安装到我自己的家目录下，而不是系统的公共目录里。
>
> `jupyter lab --no-browser --port=8889`

==利用conda直接直接建立自己的环境。==

**1. 创建属于你的虚拟环境**

Bash

```
# 创建一个名为 analysis 的环境，并安装 Python 3.10
conda create -n my_physics python=3.10
```

这里创建的环境保存在了~/.conda/envs/my_physics这里。

在 Conda 命令中，`-n` 是 **`--name`** 的缩写。

它的作用是**为你即将创建的虚拟环境指定一个名称**。

### 详细拆解：

当你运行 `conda create -n my_physics python=3.10` 时：

- **`-n`**：告诉 Conda，后面紧跟着的字符串将作为这个环境的名字。
- **`my_physics`**：这就是你给环境起的名字。以后你想进入这个环境，只需要输入 `conda activate my_physics`。

------

### 为什么要用名字（-n）而不是路径（-p）？

在 Conda 中，指定环境安装位置有两种常用方式：

| **参数** | **全称**   | **示例**          | **存储位置**                                                 |
| -------- | ---------- | ----------------- | ------------------------------------------------------------ |
| **`-n`** | `--name`   | `-n my_physics`   | 存储在你预设的默认目录中（对你来说就是 `~/.conda/envs/my_physics`）。 |
| **`-p`** | `--prefix` | `-p ./env_folder` | 存储在你当前指定的具体文件夹路径下。                         |

**使用 `-n` 的好处：**

1. **简洁**：激活环境时只需要记住名字（如 `my_physics`），而不需要输入冗长的文件夹路径。
2. **整洁**：所有的环境都会整齐地排列在你之前配置好的 `$HOME/.conda/envs` 目录下，方便统一管理。

**2. 激活环境**

Bash

```
conda my_physics analysis
```

**3. 安装分析必备库**

> .bashrc文件的作用：
>
> 简单来说，`.bashrc` 是你的 **“登录自动运行清单”**。
>
> - **自动初始化**：在 Linux 系统中，每当你通过 SSH 登录服务器或者打开一个新的终端窗口时，系统都会自动读取并执行这个文件里的命令。
> - **持久化配置**：你在终端里直接输入 `export` 或 `alias` 只能在当前会话生效，一旦断开连接就丢了。写在 `.bashrc`里，你的个人偏好（比如 `source` 某个环境）就能永久保留。

**4.利用jupyterlab，实现在浏览器里面写代码**

**第一步：在服务器端准备**

1. **安装 JupyterLab**（确保你在 `my_physics` 环境下）：

   Bash

   ```
   conda install jupyterlab
   ```

2. **启动服务**： 在服务器终端输入以下命令：

   Bash

   ```
   jupyter-lab --no-browser --port=8888
   ```

   - `--no-browser`：因为服务器没有浏览器，所以告诉它不要尝试在本地打开。
   - `--port=8888`：指定运行在 8888 端口。

3. **记录 Token**： 启动后，终端会显示一串类似 `http://localhost:8888/?token=...` 的链接。**把 `token=` 后面那长串字符复制下来**，等会儿要用。

------

**第二步：在你的 Mac 端设置 SSH 隧道**

这步最关键，它是连接你 Mac 浏览器和远程服务器的“秘密通道”。

1. **在 Mac 上打开一个新的终端窗口**（注意：不是连着服务器的那个，是本地的）。

2. **建立隧道**： 输入以下命令（请把 `sunzhongyuan` 和 `node02` 替换为你实际的登录信息）：

   Bash

   ```
   ssh -L 8889:localhost:8888 sunzhongyuan@node02
   ```

   - **这里的逻辑是**：把服务器上的 `8888` 端口，映射到你 Mac 本地的 `8889` 端口。
   - `-L` 代表 **Local Forwarding（本地转发）**。 它告诉 SSH：“请在我的 **本地（Local）** 机器上开启一个端口，并将发往该端口的所有流量，通过加密通道转发到远程服务器指定的端口上去。”
   - `8889:localhost:8888` 的拆解，这一串字符定义了流量的“起点”和“终点”：
     - **`8889` (本地端口)**：这是你 Mac 上的端口。你可以随意设置（比如 8080, 8889 等），只要不和 Mac 正在运行的程序冲突即可。你在浏览器输入 `localhost:8889` 时，就是在访问这个入口。
     - **`localhost` (目标主机)**：这里的 `localhost` 是相对于**远程服务器**而言的。它告诉服务器：“收到流量后，请把它发给**你自己**（即服务器本身）。”
     - **`8888` (目标端口)**：这是 JupyterLab 在服务器上运行的实际端口。

------

### 第三步：在 Mac 浏览器打开

1. 打开 Chrome 或 Safari，在地址栏输入： `localhost:8889`
2. **输入 Token**： 页面会提示你输入 Token，把你在第一步记录的那串字符粘贴进去。
